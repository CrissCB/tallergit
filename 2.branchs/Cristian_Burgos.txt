2.1
1. Un branch es una entorno aislado que tiene los mismos archivos del proyecto principal o la rama principal llamada main con el fin de poder trabajar sobre el proyecto sin verse afectado el proyecto que se encuentra desplegado.
2. Son utiles cuando necesitamos agregar, eliminar o modificar ya sean archivos, carpetas o codigos sin necesidad de que los usuarios o consumidores se vean afectados por estos cambios que se estan realizando hasta que se prueben y se puedan desplegar en la rama principal.
3. El comando para crear una branch es: "git branch <nombre de la rama>".
4. El comando para cambiar a una rama es: "git switch <nombre de la rama> o git checkout <nombre de la rama>".
5. El comando para eliminar una rama es: "git branch -d <nombre de la rama>".
6. El comando para cambiar y entrar a una rama es: "git checkout -b <nombre de la rama>".
7. Un merge nos ayuda a juntar una o más ramas y de esta manera sumar o restar archivos modificados haciendo un commit y dejando los cambios en una sola branch.
8. Un merge se utiliza cuando los cambios o avances del proyecto ya estan listos y probados en una rama y de esta manera pasan estos cambios funcionales a la rama principal.
9. Un tag nos permite etiquetar puntos especificos en el historial y volver facilmente a estos puntos, se utilizan comunmente para versionamientos.
10. Un tag se crea con el siguiente comando: "git tag <nombre del tag>".
